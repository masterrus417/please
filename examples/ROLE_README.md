Ролевая модель
1. компонент - аутентификация
* ввод пары логин пароль
* вызов функции верификации (пока мок)\
* обращение в бэкенд с парой, получает ответ:\
 да/нет,\
 список доступных ролей (ref_actors),\
 ключ от АПИ, с которым будет доступны остальные вызовы

2. компонент - информация о пользователе
* входящий параметр id (уникальный идентификатор пользователя) или org_no (табельный номер пользователя)
* обращение в бэкенд (пока мок), получает ответ\
		фамилия, имя, отчество, подразделение, структурная группа, должность, телефон, график работы, адрес\
		список доступных ролей (ref_actors)\
* отображение этого в компоненте

  ожидается, что этот модуль будет использоваться по клику из основного меню\
  или при просмотре информации из лога, согласовании, просмотре значений атрибута (нажал кнопку - всплыл блок с информацией)
